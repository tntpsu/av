# Triage Runbook

This runbook defines operator severity and triage workflow for gate bundles
generated by `tools/analyze/run_gate_and_triage.py`.

## Severity Model

- Sev-1 safety failure
  - Any emergency stop, out-of-lane event, or oscillation runaway in a gate run.
  - Immediate action: block promotion, generate triage packet + failure packet, and assign owner.
- Sev-2 instability or regression
  - Hard safety passes but regression budget fails (lateral metrics, comfort, or stability).
  - Immediate action: block promotion, generate triage packet with single next lever.
- Sev-3 comfort-only issue
  - Safety and regression budget pass, but comfort deltas exceed preferred targets.
  - Immediate action: allow controlled promotion only with explicit signoff.

## Quick Gate

- Scope: 2 highway + 2 s-loop runs.
- Gate outcome is read from `gate_report.json`.
- If any run fails safety/regression checks:
  - review `triage_packets/*.json`,
  - review `failure_packets/<recording_id>/`,
  - apply the single next lever from each packet.

## Promotion Gate

- Scope: 10 highway + 10 s-loop runs.
- Promotion is allowed only when:
  - hard safety gate passes all runs,
  - regression budget checks pass,
  - triage artifacts are present for every failed/regressed run.

## Operator Checklist

1. Open latest bundle under `data/reports/gates/`.
2. Read `decision.md` first.
3. If rejected, inspect triage packets in order:
   - what failed first,
   - downstream consequence,
   - single next lever.
4. Confirm diagnostics mode:
   - `to_failure` preferred,
   - `full_run_fallback` explicitly labeled when needed.
5. Only promote when `decision.json` reports `pass_fail=true`.
